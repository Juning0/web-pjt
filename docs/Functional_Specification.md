# LocalHub 기능 명세서

> 공공데이터 기반 지역 정보 공유 커뮤니티 — 대전/충청권

| 항목 | 내용 |
|------|------|
| 프로젝트명 | LocalHub |
| 선정 권역 | 대전/충청권 (대전·세종·계룡·공주·논산·옥천) |
| 문서 버전 | v1.0 |
| 최종 수정 | 2026-07-14 |

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 핵심 문제 | 지역 정보가 여러 채널에 흩어져 있고, 공공데이터만으로는 최신 현황(축제 취소, 실제 방문 후기 등)을 알기 어려움 |
| 타깃 사용자 | 대전/충청 방문 관광객 및 지역 주민 (회원가입 부담 없이 정보를 공유·소비하고 싶은 사용자) |
| 성공 지표 | 필수기능 100% 구현 + 선택기능 최소 1개 배포 완료, 챗봇 정상 응답률, 게시판 게시글/조회수 누적 |
| 서비스 특성 | 회원가입·로그인 없는 **익명 커뮤니티** |

## 4. 기능 상세 명세

### 4-1. 공공데이터 연동 (Must / BE)
- 한국관광공사 TourAPI JSON 8종(총 1,365건)을 `seed.py`로 SQLite `locations` 테이블에 적재
- 공공데이터는 **읽기 전용** — 목록/상세 조회만 제공, 수정·삭제 없음
- 카테고리: 관광지·레포츠·문화시설·쇼핑·숙박·여행코스·음식점·축제공연행사

| 기능 | 엔드포인트 | 설명 |
|------|-----------|------|
| 목록 조회 | `GET /api/locations` | 카테고리 필터, 키워드 검색, 페이지네이션 |
| 상세 조회 | `GET /api/locations/{content_id}` | 장소 정보 + 리뷰 평균 평점·리뷰 수 |

### 4-2. 익명 게시판 CRUD (Must / BE)
- 회원가입 없이 제목·내용·**수정용 비밀번호**로 게시글 작성
- 수정/삭제는 등록한 비밀번호 일치 여부로만 확인 (**평문 비교, 의도된 설계**)
- 작성 시 평점(1~5) 선택 입력, 장소별 평균 평점 노출, 평점순 정렬 옵션

| 기능 | 엔드포인트 | 비밀번호 |
|------|-----------|:---:|
| 목록 조회 (검색·정렬·페이지) | `GET /api/posts` | — |
| 상세 조회 (조회수 +1) | `GET /api/posts/{id}` | — |
| 작성 | `POST /api/posts` | — |
| 수정 | `PUT /api/posts/{id}` | ✅ |
| 삭제 | `POST /api/posts/{id}/delete` | ✅ |

> ⚠️ 비밀번호 분실 시 수정/삭제 불가 → 작성 화면에 경고 문구 노출 필요 (FE)

### 4-3. 댓글 CRUD (Must / BE)
- 게시글에 익명 댓글 작성, 비밀번호로 삭제
- 게시글 삭제 시 댓글 자동 삭제 (cascade)

| 기능 | 엔드포인트 | 비밀번호 |
|------|-----------|:---:|
| 작성 | `POST /api/posts/{post_id}/comments` | — |
| 삭제 | `POST /api/posts/comments/{comment_id}/delete` | ✅ |

### 4-4. 챗봇 (Must / 챗봇 담당)
- `POST /api/chat` — OpenAI API 활용, 제공 JSON 기반 자연어 질의응답
- 주요 질의: 관광지 추천, 축제 일정, 모범음식점 위치, 커뮤니티 게시글 검색 등
- UI: 대화 히스토리 유지, 모바일 대응, 우측 하단 플로팅 위젯

### 4-5. 게시판 부가기능 (Should / 선정)
- **조회수**: 상세 조회 시 자동 증가 (구현됨)
- **검색**: 제목·내용 키워드 검색 (구현됨)
- **좋아요**: 추후 테이블 추가 예정

### 4-6. 축제 캘린더 (Should / 선정, FE)
- FullCalendar로 권역 축제 일정 시각화
- ⚠️ **데이터 이슈**: 축제공연행사 JSON에 날짜 필드 부재 → 표시할 일정 데이터 확보 방안 선행 필요

### 4-7. 지도 마커 시각화 (Could / FE)
- Leaflet 또는 Kakao Maps로 관광지·맛집 핀 표시
- `locations.lat`/`lng` 활용, 일정 여유 시 착수

## 5. 화면 구성

| 화면 | 구성 |
|------|------|
| 홈(메인) | 권역 소개 배너, 카테고리 바로가기, 최근 게시글, 챗봇 플로팅 버튼 |

| 챗봇 위젯 | 플로팅 버튼(접힘) / 대화창(펼침), 모바일은 전체 화면 |

## 7. 제약사항

- **개발 도구**: VS Code + GitHub Copilot + OpenAI API Key 조합만 허용. Claude Code·Cursor 등 타 AI 코딩 도구 사용 금지
- **민감정보 관리**: API 키·DB 경로 등은 `.env`로 관리, `.gitignore` 등록 필수 (저장소 직접 기재 금지)
- **범위 변경**: 개발 착수 이후 기능 추가·범위 변경 원칙적 불가
- **인증 미적용**: 익명 서비스 특성상 회원가입·로그인·권한 체계 미구현. 수정/삭제 권한은 수정용 비밀번호 일치로만 확인
